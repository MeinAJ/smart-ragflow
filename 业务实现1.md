# 业务逻辑实现任务：智能问答流式接口（基于 LangGraph）

## 1. 背景

项目已搭建基础框架（`smart-ragflow`），包含 FastAPI 应用、LangGraph 占位图、环境配置等。现需实现核心问答功能：接收用户问题，经检索增强生成（RAG）后，以
SSE 流式返回答案。

## 2. 功能需求

- **HTTP 接口**：`POST /v1/chat/completions`，支持 `text/event-stream` 响应（流式输出）。
- **输入格式**：JSON，包含 `messages` 字段（如 `{"messages": [{"role": "user", "content": "问题"}]}`），可选 `stream` 参数（默认为
  `true`）。
- **输出格式**：符合 OpenAI 风格的 SSE 流式数据块（`data: {"choices": [{"delta": {"content": "文本"}}]}`）。
- **处理流程**：
    1. 用户问题通过 BGE-M3 HTTP 接口向量化为 1024 维向量。
    2. 使用向量 + 问题文本在 OpenSearch 中进行混合检索（BM25 + 向量相似度），返回 Top 10 文档块。
    3. 对检索结果执行 MMR（最大边际相关性）重排序，取 Top 5。
    4. 将重排序后的文档块与原始问题组装成 Prompt，调用 LLM（OpenAI 兼容 API）生成答案。
    5. 将 LLM 生成的答案以流式形式返回给客户端。

#### 状态定义（`state.py`）

```python
from typing import TypedDict, List, Optional
from langgraph.graph import StateGraph


class GraphState(TypedDict):
    question: str  # 原始问题
    question_vector: List[float]  # 向量化结果
    retrieved_docs: List[dict]  # 检索到的文档块（带文本、元数据）
    reranked_docs: List[dict]  # MMR 重排序后的文档块
    answer: Optional[str]  # 最终生成的答案（流式时可能不存储）
    error: Optional[str]  # 错误信息
```

#### 节点函数（nodes.py）

```
每个节点接收 state 并返回部分更新（使用 StateGraph 的 add_node）。
vectorize_node：调用 embedding 服务将 state["question"] 转为向量，存入 state["question_vector"]。
retrieve_node：使用 state["question"] 和 state["question_vector"] 调用 OpenSearch 混合检索，将结果存入 state["retrieved_docs"]。
rerank_node：对 state["retrieved_docs"] 执行 MMR，存入 state["reranked_docs"]。
generate_node：组装 Prompt（使用 reranked_docs 作为上下文），调用 LLM 流式生成，通过回调将 token 实时推送到 SSE 流。
```

#### 流式生成实现

```
使用 FastAPI 的 StreamingResponse，媒体类型为 text/event-stream。
在 generate_node 中，通过 yield 或回调函数将 LLM 的 token 逐块发送。
由于 LangGraph 节点通常返回静态结果，流式输出需突破节点范式：可以在图外部包装一个生成器函数，按顺序执行节点，当进入 generate_node 时，直接接管流式输出。
```

#### 外部服务封装

```
Embedding：调用 BGE-M3 HTTP 接口（假设地址为 EMBEDDING_URL），POST JSON {"text": "..."}，返回向量列表。
OpenSearch：使用 opensearch-py 客户端，实现混合查询：query 为 bool 组合 match（BM25）和 knn（向量搜索），size=10。
LLM：使用 httpx 或 openai SDK，配置 base_url 为 OpenAI 兼容地址，启用 stream=True，逐块解析。
```

## 4. 代码风格与注意事项

```
所有模块通过 from ... import ... 导入。
异步优先：HTTP 客户端、数据库操作均使用异步库（如 httpx.AsyncClient、opensearch-py 的异步客户端）。
错误处理：每个节点应捕获异常并设置 state["error"]，图执行时若出现错误则提前结束。
流式输出需注意编码与格式，确保符合 OpenAI API 规范（如 data: {...}\n\n）。
日志记录：使用 logging 记录关键步骤（向量化耗时、检索结果数等）。
测试：不要求编写测试，但需保证代码可运行（依赖服务可 mock）。
配置信息：不要硬编码，必须从.env配置中获取。
```
