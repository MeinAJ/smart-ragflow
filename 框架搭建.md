# 项目搭建任务：仅生成框架（无业务逻辑）

## 1. 环境初始化

- 使用 `uv` 创建虚拟环境并安装基础依赖（依赖项见 `pyproject.toml`）。
- 项目根目录命名为 `smart-ragflow`。

## 2. 目录结构（按以下结构创建文件和文件夹）

```
smart-ragflow/
├── client/
│ ├── init.py
│ ├── main.py
│ └── graph.py # LangGraph 工作流定义（空节点）
├── .env.example # 环境变量模板
├── .gitignore # 忽略 .venv, .env, pycache 等
├── pyproject.toml
├── uv.lock # 由 uv 自动生成（可留空）
└── README.md
```

## 3. 文件内容要求

### `pyproject.toml`

- 使用 `uv` 管理依赖，包含以下核心包（版本留空或使用最新稳定版）：
    - `fastapi`
    - `uvicorn[standard]`
    - `langchain`
    - `langgraph`
    - `pydantic`
    - `pydantic-settings`
    - `opensearch-py`
    - `redis`
    - `minio`
    - `python-dotenv`
    - `deepdoc`
- **包配置**：明确列出包路径 `["client"]`，**不使用** `[tool.setuptools.packages.find]`。
- 定义项目元数据（名称、版本等）。

### `client/__init__.py`

- 空文件，仅用于标记 Python 包。

### `client/main.py`

- 创建 FastAPI 应用实例。
- 添加一个根路由 `GET /` 返回 `{"status": "ok"}`（仅用于健康检查）。
- 添加一个占位路由 `GET /health` 返回同样内容。
- 使用 `uvicorn` 启动（if __name__ == "__main__" 块）。
- **不实现任何业务逻辑**（如文档上传、检索、对话等）。

### `client/graph.py`

- 导入 `langgraph` 相关组件。
- 定义一个简单的状态图（例如两个节点，一个入口，一个结束），所有节点函数内部只写 `pass` 或 `return {}`。
- 返回编译后的图对象，用于后续扩展。

### `.env.example`

- 列出所有必需的环境变量（无敏感值，仅示例）：

```
OPENAI_API_KEY=your-key-here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENSEARCH_HOST=localhost
OPENSEARCH_PORT=9200
REDIS_URL=redis://localhost:6379
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin
EMBEDDING_MODEL=BGE-M3
```

- 添加注释说明每个变量的用途。

### `.gitignore`

- 包含常见 Python 忽略项：

```
.venv/
.env
pycache/
*.pyc
.pytest_cache/
*.log
```

### `README.md`

- 简要说明项目基于的技术栈。
- 包含快速启动步骤（`uv sync`, `uv run uvicorn client.main:app --reload`）。
- 说明当前仅为框架，业务逻辑待实现。

## 4. 代码风格与规范

- 所有 Python 文件使用 `from ... import ...` 导入，避免相对导入。
- 函数、类均添加 docstring（简要说明，如“占位节点”）。
- 任何可能包含业务逻辑的地方（如 LLM 调用、向量检索、缓存操作）均用 `pass` 或 `return None` 占位，并加上 `# TODO: 实现业务逻辑`
  注释。

## 5. 其他要求

- 项目必须可运行（`uv run uvicorn client.main:app --reload` 能正常启动且不报错）。
- 不实现任何与“智能问知”相关的具体功能（如对话、检索、文档处理）。
- 框架需严格遵循“技术栈选型”中列出的组件，但所有外部服务（OpenSearch、Redis、Minio、BGE-M3）的连接代码仅做初始化配置，不实际调用其业务方法。
